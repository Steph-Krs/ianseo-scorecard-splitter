# Ianseo Scorecard Splitter

🏹 **Ianseo Scorecard Splitter** is a Python script to split large, multi-page PDF scorecard files generated by [Ianseo](https://www.ianseo.net/) into individual, uniquely-named PDF files for each archer, based on the scorecard content (target position and barcode). Additionally, it can now optionally redact target position tokens on each page (e.g., 97A/97B/97C/97D) before generating PDFs.

Ianseo only allows exporting a single multi-page PDF without automated unique naming for each archer — this tool fills that gap.

---

## 🚀 Features

✅ Splits one PDF per archer from the large multi-page PDF  
✅ Names each file as `<position>_<barcode>.pdf` (optionally prefixed with club ID using `-c`)  
✅ Compresses the output PDFs to save space while preserving image quality (300 dpi)  
✅ Supports batch processing of multiple PDFs  
✅ Displays a progress bar during processing  
✅ Optional redaction of target positions (e.g., 97A/97B/97C/97D) using `-r`  
✅ Supports grouping all generated PDFs by club into a single folder using `-g`

## 📄 Usage

Basic usage:
```bash
python Ianseo-Scorecard-Splitter.py <file1.pdf> <file2.pdf> ...
```

Optional flags:
- `-c` / `--include-club` : prefix filenames with detected club ID (7 alphanumeric chars)  
- `-g` / `--group-by-club` : group generated pages by club into per-club PDFs in a single folder (removes intermediate files)  
- `-r` / `--remove-position` : redact the four position tokens (e.g., 97A/97B/97C/97D) from each page before saving
- `-o` / `--output-root` : specify root directory for output folders  
- `--color-dpi` / `--gray-dpi` : control Ghostscript compression DPI (default: 300)

Example with position removal and club grouping in the user directory with a resolution of 600 dpi:
```bash
python Ianseo-Scorecard-Splitter.py doc1.pdf doc2.pdf -c -g -r -o /home/user/ --color-dpi 600
```

---

## 📁 Output

- Each PDF is named based on the archer’s position and barcode, optionally prefixed with club ID, for example:
```
1234567_97A_1234567A-CL-U21-1.pdf
```
- When using `-g`, all club PDFs are stored in a single `club_pdfs` folder.  
- Intermediate single-page folders are automatically removed after grouping.

---

## 🔷 Notes

- Redaction of positions is performed on temporary PDFs before compression; the overlay color is light gray (#E8E8E8) and padding can be adjusted in the script.  
- The Regex rules for position, club ID, and barcode can be modified in lines 46-48 of the script if necessary.  
- Works best on Ubuntu 24.04 or similar Linux environments.  
- Run times can be long for very large PDFs; the progress bar shows page processing progress.
