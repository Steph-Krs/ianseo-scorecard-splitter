# Ianseo Scorecard Splitter

🏹 **Ianseo Scorecard Splitter** is a Python script to split large, multi-page PDF scorecard files generated by [Ianseo](https://www.ianseo.net/) into individual, uniquely-named PDF files for each archer, based on the scorecard content (target position and barcode). Additionally, it can optionally redact target position tokens on each page (e.g., 1205A/1205B/1205C/1205D/1205E/1205F/1205G/1205H) before generating PDFs.

Ianseo only allows exporting a single multi-page PDF without automated unique naming for each archer — this tool fills that gap.

---

## 🚀 Features

✅ Splits one PDF per archer from the large multi-page PDF  
✅ Names each file as `<position>_<barcode>.pdf` (optionally prefixed with club ID using `-c`)  
✅ Compresses the output PDFs to save space while preserving image quality (300 dpi by default)  
✅ Supports batch processing of multiple PDFs with **parallel processing** for faster execution  
✅ Displays a progress bar during processing  
✅ Optional redaction of target positions (e.g., 1205A/1205B/1205C/1205D/1205E/1205F/1205G/1205H) using `-r`  
✅ Supports grouping all generated PDFs by club into a single folder using `-g`  
✅ Debug mode (`-d`) to generate detailed logs and statistics about skipped pages  
✅ Supports up to 9999 targets per session with positions A-H (8 archers per target)  
✅ Multi-core CPU utilization for significantly faster processing times

## 📄 Usage

### Quick Start Commands

**Before running the script each time**, activate the virtual environment:
```bash
cd ~/ianseo-scorecard-splitter
source venv/bin/activate
```

**Basic usage** (process PDFs with all features):
```bash
python Ianseo-Scorecard-Splitter.py <file1.pdf> <file2.pdf> ... -c -g -r
```

**Recommended usage** (with debug mode for diagnostics):
```bash
python Ianseo-Scorecard-Splitter.py <file1.pdf> <file2.pdf> ... -c -g -r -d
```

**Process all PDFs in current directory**:
```bash
python Ianseo-Scorecard-Splitter.py *.pdf -c -g -r
```

**When finished**, deactivate the virtual environment:
```bash
deactivate
```

---

### Available Flags

- `-c` / `--include-club` : Prefix filenames with detected club ID (7 alphanumeric chars)  
- `-g` / `--group-by-club` : Group generated pages by club into per-club PDFs in a single folder (removes intermediate files)  
- `-r` / `--remove-position` : Redact the eight position tokens (e.g., 1205A/1205B/1205C/1205D/1205E/1205F/1205G/1205H) from each page before saving  
- `-d` / `--debug` : Enable debug mode to create detailed log files showing which pages were skipped and why  
- `-o <dir>` / `--output-root <dir>` : Specify root directory for output folders (defaults to current directory)  
- `-j <N>` / `--jobs <N>` : Number of parallel processes to use (default: automatically uses all CPU cores). Use `-j 1` to disable parallelization  
- `--color-dpi <N>` : Ghostscript color image DPI for compression (default: 300)  
- `--gray-dpi <N>` : Ghostscript gray image DPI for compression (default: 300)

---

### Usage Example

Example with position removal and club grouping in the user directory, with a resolution of 600 dpi, with debug mode, with 2 CPU used :
```bash
python Ianseo-Scorecard-Splitter.py doc1.pdf doc2.pdf -c -g -r -o /home/user/ --color-dpi 600 -d -j 2
```

## 📁 Output

### File Naming

Each PDF is named based on the archer's position and barcode, optionally prefixed with club ID:

**Without `-c` flag**:
```
1205A_1234567E-CL-FlBla-1.pdf
```

**With `-c` flag**:
```
1234567_1205A_1234567E-CL-FlBla-1.pdf
```

### Output Structure

**Without `-g` flag** (individual files per input PDF):
```
doc1_pdfs/
  ├── 1205A_1234567E-CL-FlBla-1.pdf
  ├── 1205B_1234567E-CL-FlNoi-1.pdf
  └── ...
doc2_pdfs/
  ├── 1206A_1234568E-CL-FlBla-1.pdf
  └── ...
```

**With `-g` flag** (grouped by club):
```
club_pdfs/
  ├── 1234567.pdf  (contains all pages for club 1234567)
  ├── 1234568.pdf  (contains all pages for club 1234568)
  └── ...
```

When using `-g`, intermediate single-page folders are automatically removed after grouping.

## 🔍 Debug Mode

Enable debug mode with `-d` to generate detailed diagnostic information:

### Debug Log Files

For each input PDF, a log file is created: `<pdf_name>_pdfs/debug_<pdf_name>.log`

**Log file contains**:
- Position, barcode, and club ID detected for each page
- Status (OK or SKIPPED) for each page
- Text samples from skipped pages
- Summary statistics

### Console Statistics

When debug mode is enabled, the script displays:
- Total pages processed vs. successful
- Breakdown of why pages were skipped:
  - No position found
  - No barcode found
  - Both missing
  - Text extraction errors
  - Save errors
- Percentage of successful pages
- Sample of first skipped pages with reasons

**Example output**:
```

--------------------------------------------------------------------------------

doc1.pdf:
  Total pages: 1056
  Successful: 1056 (100.0%)
  Skipped: 0
```

## ⚡ Performance & Parallel Processing

The script automatically uses all available CPU cores to process multiple PDFs simultaneously.

### Performance Comparison

**Sequential processing** (1 core):
- 5 PDFs, ~5300 pages total: **~40 minutes**

**Parallel processing** (5+ cores):
- 5 PDFs, ~5300 pages total: **~8-12 minutes** 🚀

### Optimization Tips

1. **Split large PDFs into smaller files** before processing
   - Optimal: Create as many PDFs as you have CPU cores
   - Example: Split into 8 smaller PDFs if you have 8 cores

2. **Check CPU usage**:
   ```bash
   # See number of CPU cores available
   nproc
   ```

3. **Control resource usage**:
   ```bash
   # Limit to 4 parallel processes to preserve system resources
   python Ianseo-Scorecard-Splitter.py *.pdf -c -g -r -j 4
   ```

## 🔷 Technical Notes

### Regex Patterns

The script uses the following regex patterns (defined in lines 42-44):

- **Position**: `\b(\d{1,4}[A-H])\b` — Matches target positions from 1A to 9999H
- **Barcode**: `(\d{7}[A-Z]-[A-Z]{2}-[A-Za-z]+-\d+)` — Matches archer barcodes (e.g., 1234567E-CL-FlBla-1)
- **Club ID**: `\b([0-9A-Z]{7})\b` — Matches 7-character club identifiers

These patterns can be modified if needed for different formats.

### Position Redaction

When using `-r`:
- Redaction is performed on temporary PDFs before compression
- Overlay color: light gray (#E8E8E8)
- All eight position tokens (A-H) are redacted for the detected base number
- Padding can be adjusted in the script (line 91)

### Compression Settings

- **Default DPI**: 300 for both color and grayscale images
- **Mono images** (barcodes/QR codes): kept lossless, no downsampling
- **Ghostscript** is used for reliable, high-quality compression

### System Requirements

- **OS**: Ubuntu 24.04 or similar Linux environments (tested on Ubuntu)
- **RAM**: ~500 MB per parallel process (e.g., 4 GB total for 8 processes)
- **CPU**: Multi-core processor recommended for best performance
- **Disk**: Temporary space for intermediate files (cleaned up automatically with `-g`)

## 🐛 Troubleshooting

### Pages are being skipped

Run with debug mode to see why:
```bash
python Ianseo-Scorecard-Splitter.py your_file.pdf -d
```

Check the debug log file and console output for:
- Missing position or barcode patterns
- Text extraction errors
- Incorrect regex patterns

### Script runs slowly

- Enable parallel processing by processing multiple PDFs at once
- Split large PDFs into smaller files (one per CPU core is optimal)
- Check system resources: `htop` or `top`

### Ghostscript compression fails

The script will fall back to uncompressed PDFs. If you see compression warnings:
- Verify Ghostscript is installed: `gs --version`
- Check disk space in output directories
